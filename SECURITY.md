# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.x     | ✅ Yes    |
| < 1.0   | ❌ No     |

## Reporting a Vulnerability

**Do NOT open a public GitHub issue.**

Email: [sandeep.int.2005@gmail.com](mailto:sandeep.int.2005@gmail.com)

Response within **48 hours**.

Include in your report:

- Description of the vulnerability
- The exact payload used
- Expected verdict vs actual verdict
- Steps to reproduce
- Impact assessment
- Any suggested fix (optional)

We take all security reports seriously. Responsible disclosure is appreciated.

If confirmed, we will patch, credit you (if desired), and update the changelog.
